# 代理列表批量导入与批量管理设计

## 背景

当前设置页的代理列表仅支持单条新增、编辑、测试、启停和删除，不适合一次性导入大量代理，也缺少列表级筛选和批量移除能力。

本次设计目标是在现有“设置 > 代理设置”页面内补齐批量导入、自动格式识别、归属地命名、列表筛选和批量删除能力，同时尽量保持现有页面结构和后端风格不变。

## 目标

- 在代理列表区域增加批量导入能力
- 自动识别并解析多种常见代理格式
- 支持无认证代理和带认证代理混合导入
- 自动查询 IP 归属地并生成代理名称
- 在无法获取归属地时回退到通用名称
- 支持按关键词、类型、启用状态、是否默认、国家/城市筛选
- 支持多选后批量删除

## 非目标

- 不新增独立导入页面或多步导入向导
- 不支持 IPv6 导入
- 不支持用户名或密码中包含 `:` 或 `@`
- 不支持导入前逐条预览和手动勾选入库
- 不将归属地查询做成可配置后台选项

## 用户确认的关键规则

- 批量导入自动命名：`国家-城市-001`
- 无法查询归属地时回退名称：`代理-001`
- 额外支持格式：`主机IP:端口`
- 无协议时，代理类型由用户在导入面板选择默认值
- 重复项处理：跳过，不覆盖已有记录
- 列表支持批量筛选和批量移除
- 主机支持 IPv4 和域名，不支持 IPv6
- 用户名和密码中不允许包含 `:` 或 `@`
- 归属地查询使用并发以提升批量导入效率

## 页面方案

功能继续放在 `templates/settings.html` 的代理设置区域，不拆新页面。

### 1. 批量导入卡片

在现有“代理列表”上方新增批量导入卡片，包含：

- 默认代理类型下拉框：`HTTP` / `SOCKS5`
- 多行文本框
- 示例格式说明
- 导入按钮
- 导入结果区：成功、跳过、失败统计和失败明细

### 2. 列表筛选栏

在代理表格上方增加筛选控件：

- 关键词
- 类型
- 启用状态
- 是否默认
- 国家/城市

筛选方式采用数据库条件查询，结果直接刷新代理表格。

### 3. 批量操作

在代理列表增加：

- 每行复选框
- 表头全选
- 批量删除按钮

批量删除前弹出确认提示，执行后刷新列表并清空选中态。

## 支持的导入格式

首期支持以下 5 种格式：

1. `协议://主机IP:端口:用户名:密码`
2. `主机IP:端口:用户名:密码`
3. `用户名:密码@主机IP:端口`
4. `主机IP:端口@用户名:密码`
5. `主机IP:端口`

其中：

- `协议` 仅接受 `http` 和 `socks5`
- 无协议时使用导入面板中的默认类型
- `主机IP` 实际支持 IPv4 和域名
- 空行和 `#` 开头的注释行跳过，不记为失败

## 解析设计

### 统一解析顺序

为减少歧义，每行按固定优先级识别：

1. `协议://主机:端口:用户名:密码`
2. `用户名:密码@主机:端口`
3. `主机:端口@用户名:密码`
4. `主机:端口:用户名:密码`
5. `主机:端口`

每行只允许命中一种格式，命中后转成统一结构：

- `line_no`
- `raw_line`
- `type`
- `host`
- `port`
- `username`
- `password`

### 校验规则

- 端口必须在 `1-65535`
- 协议仅允许 `http` / `socks5`
- 主机必须是合法 IPv4 或合法域名
- 用户名和密码中不得包含 `:` 或 `@`
- 格式错误、端口非法、协议非法、主机非法都记为失败

## 去重规则

重复判定仅基于 `host + port`：

- 如果数据库已存在相同 `host:port`
- 本次导入该行记为 `skipped`
- 不覆盖原有类型、认证、名称、状态、归属地信息

此规则与用户预期保持一致，即“同一个 IP:端口视为同一代理”。

## 归属地查询设计

### 查询来源

参考 `ihmily/ip-info-api` 项目中的免费查询源清单，首期接入两级 Provider：

1. 主 Provider：IP.SB
2. 备用 Provider：FreeIPAPI

归属地查询只作为增强信息，不作为导入成功前提。

### 查询策略

- 对 IPv4 直接查询
- 对域名先解析 A 记录，再查询首个可用 IPv4
- 单个 Provider 失败后自动切备用 Provider
- 全部失败时返回空归属地，不阻断导入

### 并发策略

归属地查询采用“串行解析，有限并发查询”的方式：

1. 先串行完成逐行解析、校验和去重
2. 提取需要查询归属地的唯一主机集合
3. 使用线程池并发查询归属地
4. 主线程汇总结果、生成名称并写库

并发规则：

- 只对归属地查询使用线程池
- 默认 `max_workers = 8`
- 每个请求批次内对相同 IP 做内存缓存
- 为每个外部请求设置短超时

这样可以提升导入速度，同时避免多线程直接操作数据库和名称编号导致的数据竞争。

## 名称生成规则

名称由独立生成器统一处理：

- 同时拿到国家和城市：`国家-城市-001`
- 任一字段缺失或查询失败：`代理-001`

编号策略：

- 以前缀为维度查找现有代理名称中的最大编号
- 新代理使用 `最大编号 + 1`
- 名称生成与入库统一在主线程完成，避免并发下重复编号

归属地原始字段单独存库，不从名称反向解析。

## 数据模型变更

扩展 `src/database/models.py` 中的 `Proxy` 模型：

- 新增 `country: Optional[str]`
- 新增 `city: Optional[str]`

对应变更：

- `Proxy.to_dict()` 增加 `country` 和 `city`
- 初始化数据库/迁移逻辑补齐新字段
- CRUD 查询支持基于这两个字段筛选

单独存储归属地字段有两个好处：

- 代理名称允许后续手工修改，不影响筛选
- 不需要再从显示名称中反向推断国家和城市

## 后端组件拆分

建议新增或抽离 4 类职责明确的组件：

### 1. 代理行解析器

职责：

- 识别多种代理文本格式
- 标准化输出统一代理结构
- 完成基础字段校验

### 2. 归属地查询器

职责：

- 根据 IPv4 或域名解析出的 IPv4 查询归属地
- 按 Provider 链失败回退
- 在单批导入中做结果缓存

### 3. 名称生成器

职责：

- 根据 `country/city` 生成标准名称前缀
- 分配递增编号

### 4. 批量导入服务

职责：

- 串起解析、去重、归属地查询、命名和入库
- 输出整批导入统计和逐行结果

## API 设计

### 1. 批量导入接口

新增：

- `POST /settings/proxies/batch-import`

请求体：

- `data: str` 多行代理文本
- `default_type: "http" | "socks5"`

返回：

- `total`
- `success`
- `skipped`
- `failed`
- `results[]`

每条 `results` 包含：

- `line_no`
- `raw_line`
- `status` (`success` / `skipped` / `failed`)
- `message`
- 成功时返回解析后的核心字段和生成名称

### 2. 代理列表筛选接口

复用现有：

- `GET /settings/proxies`

新增查询参数：

- `keyword`
- `type`
- `enabled`
- `is_default`
- `location`

筛选规则：

- `keyword` 匹配名称、主机
- `location` 同时匹配国家和城市

### 3. 批量删除接口

新增：

- `POST /settings/proxies/batch-delete`

请求体：

- `ids: number[]`

返回：

- `requested`
- `deleted`
- `missing`

不存在的 ID 忽略，不作为整体失败。

## 前端交互细节

### 批量导入

前端不做复杂预解析，仅负责：

- 收集导入文本和默认类型
- 调用批量导入接口
- 展示统计结果和失败明细
- 导入成功后刷新代理列表

### 筛选

筛选栏采用轻量联动：

- 修改条件后点击筛选按钮，或在实现成本可控时支持防抖自动刷新
- 清空筛选后恢复完整列表

### 批量删除

- 仅当存在选中项时激活批量删除按钮
- 删除成功后提示实际删除数量
- 删除后刷新列表并清空全选状态

## 异常处理

### 导入阶段

- 空文本：直接提示用户输入内容
- 单行解析失败：记录失败，不影响其他行
- 归属地接口超时或异常：回退默认名称，继续导入
- 数据库单行写入异常：记录失败并继续处理下一条

### 筛选阶段

- 条件为空时返回全部数据
- 无匹配结果时显示空状态，不报错

### 批量删除阶段

- 部分 ID 不存在时忽略并返回统计
- 全部不存在也返回成功响应，但 `deleted = 0`
