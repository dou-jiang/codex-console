# Yahoo 母号临时子地址注册 OpenAI 使用说明

## 先回答两个常见问题

### 1. 为什么页面里会出现两个“注册类型”？

原因不是业务上真的需要两个，而是前端模板里重复写了两套同名控件：

- 两个都叫“注册类型”
- 两个都用了同一个 `id="registration-type"`
- 两个都用了同一个 `name="registration_type"`

这会导致两个问题：

1. 页面上肉眼看到两个“注册类型”下拉框
2. JavaScript 通过 `document.getElementById('registration-type')` 只会拿到其中一个，界面和真实提交值可能不一致

当前已经修正为只保留 **一套** 注册类型控件。

### 2. 基于 Yahoo 邮箱，注册类型应该怎么选？

Yahoo 邮箱服务本身只负责“收验证码”，不决定 OpenAI 账号是母号还是子号。  
`注册类型` 的含义是你希望这个 OpenAI 账号在系统里如何标记。

建议如下：

- `普通`
  - 默认推荐
  - 如果你只是想先跑通 Yahoo alias 注册链路，选这个最稳妥

- `母号`
  - 只有当你明确要把这个 OpenAI 账号作为 Team/管理账号使用时才选

- `子号`
  - 当你明确要把它作为成员账号、候选子账号使用时再选

如果你当前目标只是验证 Yahoo 母号 -> 临时子地址 alias -> OpenAI 注册是否能跑通，建议先选：

`普通`

## 功能目标

当前 Yahoo 流程已经调整为更贴近 Yahoo Mail Plus 的真实用法：

1. 使用一个 `Yahoo 母号` 作为主收件箱
2. 在母号里自动创建一个 `临时子地址 alias`
3. 使用这个 alias 去注册 OpenAI
4. OpenAI 发给 alias 的验证码最终仍由 `母号收件箱` 接收
5. 系统在母号收件箱中按 `alias 地址` 过滤邮件并提取验证码

也就是说，当前默认不是“创建独立 Yahoo 子账号”，而是“由母号创建临时子地址 alias”。

## 实际注册流程

### 默认流程

1. 登录母号 Yahoo 邮箱
2. 进入 `Yahoo 便捷邮箱 / 临时邮件地址` 页面
3. 自动创建一个新的临时子地址 alias
4. 使用这个 alias 提交 OpenAI 注册
5. OpenAI 验证码发送到 alias
6. Yahoo 母号收件箱收到这封邮件
7. 系统在母号收件箱中按 alias 地址过滤并提取验证码
8. 完成 OpenAI 注册

### 兼容流程

如果你显式配置了：

- `固定子邮箱地址`
- `固定子邮箱密码`

则系统仍可以走兼容模式，直接复用这个固定 Yahoo 邮箱，而不是自动创建 alias。

## 两条候选流程的验证结论

### 1. `母号创建后，再去母号自动转发里新增子邮箱，然后由母号接收验证码`

这条流程 **不可行，也不是当前实现**。

原因：

- Yahoo alias 模式下，邮件本来就是进入 `母号收件箱`
- `自动转发` 不是 alias 创建入口，也不是当前代码实现链路的一部分
- 当前系统没有实现“进入自动转发设置并新增目标邮箱”的自动化流程

结论：

- 默认流程不需要自动转发
- 当前注册链路不依赖自动转发功能

### 2. `母号创建后得到子邮箱，然后登录子邮箱接收验证码`

这条流程只在 **兼容模式** 下成立一部分：

- 如果你填写的是 `固定 Yahoo 邮箱/固定子邮箱`，系统可以直接复用这个邮箱收码
- 如果你走的是 `母号创建临时子地址 alias` 模式，这个 alias 不是独立登录邮箱，不能再单独登录“子邮箱”收码

结论：

- `alias 主流程`：不支持“登录子邮箱收码”
- `固定 Yahoo 邮箱兼容模式`：支持直接复用该邮箱收码

## 子地址资料自动生成

当前子地址资料支持自动生成，默认不需要你手工填写固定模板。

下面这些字段可以直接留空：

- `名字`
- `姓氏`
- `出生月份`
- `出生日期`
- `出生年份`
- `用户名/前缀`

留空时系统会自动生成更像真人的资料，生成逻辑包括：

- 常见英文姓名组合
- 合理成人年龄区间
- 结合姓名和随机数字生成前缀

如果你手动填写了这些字段，则优先使用手填值。

## 配置项说明

### 母号 Yahoo 信息

- `母号 Yahoo 邮箱`
  - 必填
  - 用于登录 Yahoo 并承接 alias 邮件

- `母号登录密码`
  - alias 主流程必填
  - 用于浏览器登录母号并创建临时子地址 alias

- `母号 IMAP 授权码`
  - 强烈推荐
  - 用于通过 IMAP 读取母号收件箱里的 OpenAI 验证码邮件
  - 这是当前最稳定的取码方式

### 固定子邮箱配置

只有兼容模式才需要：

- `固定子邮箱地址`
- `固定子邮箱密码`
- `固定子邮箱 IMAP 授权码`

如果不填固定子邮箱，系统默认走“母号自动创建临时子地址 alias”流程。

### 子地址资料

这些字段都可以留空自动生成：

- `名字`
- `姓氏`
- `出生月份`
- `出生日期`
- `出生年份`
- `用户名/前缀`

可选辅助字段：

- `注册手机号`
- `备用恢复邮箱`

## 推荐配置

建议至少填写：

- `母号 Yahoo 邮箱`
- `母号登录密码`
- `母号 IMAP 授权码`
- `无头浏览器 = true`

子地址资料字段建议全部留空，让系统自动生成。

## 页面操作步骤

### 新增 Yahoo 服务

1. 打开“邮箱服务”页面
2. 点击“添加服务”
3. 服务类型选择 `Yahoo Mail（母号 alias）`
4. 填写母号信息
5. 子地址资料字段可留空
6. 保存服务
7. 使用“测试”先检查基础连接

### 使用 Yahoo 注册 OpenAI

1. 打开“注册”页面
2. 邮箱服务选择刚创建的 Yahoo 服务
3. 注册模式按需选择
4. `注册类型` 优先选 `普通`
5. 点击“开始注册”

系统会自动：

- 登录母号
- 创建临时子地址 alias
- 用 alias 提交 OpenAI 注册
- 再从母号收件箱按 alias 过滤验证码

## 测试与验证结果

当前已经完成的开发验证包括：

1. `yahoo_mail` 服务类型已注册
2. Yahoo 服务元数据已包含母号字段
3. 子地址资料自动生成逻辑可正常产出非空姓名与合理生日
4. alias 模式下，取验证码时会使用：
   - `母号邮箱` 作为真实收件箱
   - `子地址 alias` 作为过滤目标

## 注意事项

### 1. alias 创建依赖页面结构

Yahoo 设置页、`Yahoo 便捷邮箱`、`临时邮件地址` 的 DOM 一旦变化，浏览器自动化就可能失效。

### 2. IMAP 仍然是最关键的稳定项

如果母号没有配置 `IMAP 授权码`，系统就只能回退到浏览器轮询收件箱，稳定性会明显下降。

### 3. 遇到风控不会绕过

当前实现不会绕过：

- CAPTCHA
- challenge
- 强制手机验证

一旦 Yahoo 或 OpenAI 命中这类页面，流程会直接中止并报错。

## 使用建议

- 第一次联调时建议把 `无头浏览器` 设为 `false`，先观察是否能成功进入 alias 设置页
- 稳定后再切回 `true`
- 优先保证母号 IMAP 可用
- 子地址资料尽量留空，直接使用自动生成逻辑
- 如果只是验证 Yahoo 链路，`注册类型` 先选 `普通`
